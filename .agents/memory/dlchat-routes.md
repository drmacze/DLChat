# DLChat AI & Streak Routes

## DB Tables (added to api-server/src/index.ts runMigrations)
- `streaks` — (user_id PK, current_streak, longest_streak, last_active_date, updated_at)
- `ai_contacts` — (id TEXT PK, user_id, name, country, gender, persona_json, created_at, updated_at)
- `ai_messages` — (id UUID PK, user_id, ai_contact_id, role, content, mood, created_at)

## Routes (registered in routes/index.ts)
- `router.use("/ai", aiRouter)` — from `routes/ai.ts`
- `router.use("/streak", streakRouter)` — from `routes/streak.ts`

## API Endpoints
- `GET /api/ai/contacts` — seeds 6 default personas on first call, returns all for user
- `GET /api/ai/contacts/:aiId/messages` — message history
- `POST /api/ai/contacts/:aiId/messages` — send message, get AI reply with `typingMs` delay hint
- `DELETE /api/ai/contacts/:aiId` — remove AI contact
- `GET /api/streak` — returns + updates streak; call on every app open

**Why:** These routes and tables are not generated by Drizzle ORM — they're raw SQL in runMigrations and Express routes. Keep them in sync manually if schema changes.
